一、MySql索引

1.1 索引是什么

  • 索引是帮助MySql高效获取数据的排好序的数据结构

    • 在没有索引的时候是磁盘寻址,查询数据的时候是一个一个查找,在大数据查找时候会 很慢.

    • 为了加快查找效率,可以维护一个二叉查找树,每个节点分别包含索引键值和一个指向对相应数据记录物理地址的指针,这样就可以运用二叉树快速获取到响应数据

  • 索引数据结构

    • 二叉树

      • 如果数据自增的话,二叉树就变成链表了,高度会无限增高,查询效果会跟链表相似

    • 红黑树

      • 特殊的二叉树,简称二叉平衡树,高度还是不确定

    • BTree

      • 多路平衡搜索树

      • 叶节点具有相同深度,叶节点的指针为空

      • 所有索引元素不重复

      • 节点中的数据索引从左到右递增排列

    • B+Tree

      • 优化BTree（非叶子节点：索引+指针、叶子节点：索引+数据【地址】）

      • 非叶子节点不存储data,只存储索引(冗余),可以放更多的索引

      • 叶子节点包含所有索引字段

      • 叶子节点使用指针来凝结,提高区间访问的性能,

      • 相当于一个双向列表.

    • Hash:做哈希运算,得到散列值,再从中快速定位所需要的数据.一一映射.,只需要进行一次哈希运算就可以快速定位,所以查找速度非常快,哈希之对于精确查找,对于范围查找不行,因为有范围,不好算哈希值.

      • MD5就是一种哈希算法

二、MySql存储引擎

MySQL存储引擎的不同，那么索引文件保存的方式也有所不同，常见的有二种存储引擎MyISAM和InnoDB。

根节点常驻内存

2.1 MyISAM（非聚集索引）

MySQL5.5版本之前默认存储引擎

特点：不支持事务、不支持外键约束

2.2 InnoDB（聚集索引）

  • innoDB索引实现(聚集)

    • 表数据文件按照T+TREE组织的一个索引结构文件

    • 聚集索引-叶节点包含了完整的数据巨鹿

    • 为什么innoDB表必须有主键,并且推荐使用整型的自增主键,

    • 为什么非主键索引结构叶子节点存储的是主键值?(一致性和节省存储空间)

MySQL5.5版本之后默认存储引擎

特点：支持事务、支持外键约束
